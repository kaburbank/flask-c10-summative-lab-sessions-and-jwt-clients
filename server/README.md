# Session-Based Flask API

This backend implements:
- User signup/login/logout using Flask sessions
- Secure password hashing with Flask-Bcrypt
- Database migrations with Flask-Migrate
- A user-owned `notes` resource
- Protected CRUD endpoints for notes
- Pagination on the notes index route

## Project Structure

- `app.py` - Flask app factory and RESTful route handlers
- `models.py` - SQLAlchemy models (`User`, `Note`)
- `extensions.py` - shared extension instances (`db`, `migrate`, `bcrypt`)
- `seed.py` - seed script for sample users/notes
- `migrations/` - Alembic migration scripts generated by Flask-Migrate
- `requirements.txt` - backend dependencies

## Endpoints

### Auth
- `POST /signup`
- `POST /login`
- `GET /check_session`
- `DELETE /logout`

### Notes
- `GET /notes?page=1&per_page=10`
- `POST /notes`
- `PATCH /notes/<note_id>`
- `DELETE /notes/<note_id>`

All notes routes require a valid logged-in session and only expose notes for the current user.

## Setup and Run

```bash
cd server
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Migrations (Required)

```bash
cd server
export FLASK_APP=app.py
flask db upgrade
```

If this is the first time setting up migrations in a fresh clone:

```bash
cd server
export FLASK_APP=app.py
flask db init
flask db migrate -m "initial schema"
flask db upgrade
```

## Seed Data

```bash
cd server
python seed.py
```

## Automated Tests

Run the lightweight backend test suite with one command:

```bash
cd server
pytest
```

## Start Server

```bash
cd server
python app.py
```

The API runs on `http://localhost:5555`.
